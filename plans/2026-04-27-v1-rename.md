## Risk / trade-offs

- **No CLI alias for `-Family`**: users who scripted `bench -Family X`
  break on upgrade. We accept this; pre-v1 = no stability guarantee.
  Release notes call it out.
- **Migration of cached results**: in-place rewrite. Loses no data
  (we only add fields and remove the old `family`). Safe even if
  interrupted (each file is rewritten atomically via PS `Out-File`
  on a per-file basis).
- **`series` parsing rule**: regex-based, may misclassify exotic
  filenames (e.g. `Phi-3.5-MoE`). The fallback is `series = model`
  (unchanged), so the worst case is "the new column is the same as
  the model name" â€” not a crash, just a missed grouping opportunity.
  Edge cases get a `dense_overrides`-style override list later if
  needed.
- **Diff size**: ~30 files touched. Mitigated by atomic commits.
