# Spec: restore strict semantic versioning

## Goal

Remove the "while the project is offline, breaking changes may ship
as minor bumps" allowance from the project's versioning convention.
The repository is now public; the condition that justified the
allowance no longer holds.

## Background

`architecture/README.md`, under "Pre-public flexibility", currently
states: while the project is offline (no public remote, no external
users, no package registry), breaking changes are allowed at minor
bumps even after v1.0.0; once the project is published, strict
semantic versioning applies.

The repository is now public at `github.com/SpeederX/calibr`. The
condition is met. The allowance comes out.

## Behavior

### `architecture/README.md`

The Versioning section is edited as follows:

- The "Pre-public flexibility" subsection is removed in full.
- A single replacement sentence is added at the end of the
  Versioning section:

  > Strict semantic versioning applies. Every breaking change to the
  > command-line interface, the configuration schema, or the output
  > format is a major bump.

- The table that maps bump type to trigger is untouched.

### `memories/memory-2.md`

A single dated note is appended at the bottom of the "What changed
in v1.0.0" section, after item 3:

> Note (added 2026-05-13): the relaxed-versioning allowance applied
> to v1.0.0 only. From v1.0.1 onward, strict semantic versioning is
> in effect; see `memories/memory-3.md`.

This is the only edit to `memory-2.md`. The rest of the file is
historical and stays as written.

### Other files

No other file is touched by this spec. References to the relaxation
that may have leaked into other documentation are addressed by
`spec/v1.0.1-docs-and-roadmap-sync.md`.

## Acceptance

- [ ] `git grep -n "Pre-public flexibility"` returns nothing.
- [ ] `git grep -n "while offline"` returns nothing in
      `architecture/README.md`.
- [ ] `git grep -n "while the project is offline"` returns nothing
      in `architecture/README.md`.
- [ ] `architecture/README.md` Versioning section ends with the
      replacement sentence above.
- [ ] `memories/memory-2.md` contains the dated note recording that
      the allowance applied to v1.0.0 only.

## Out of scope

- Any retroactive re-tagging of v1.0.0. The v1.0.0 release used the
  allowance correctly under the rules in force at the time and is
  not revisited.
- Definition of what counts as a breaking change beyond what the
  existing table in `architecture/README.md` already says.
