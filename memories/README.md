# Memories

> ⚠ **ARCHIVED.** This folder used to hold point-in-time
> state-of-the-project snapshots, one per significant transition
> (`memory-1.md`, `memory-2.md`, `memory-3.md`). The cadence was
> dropped when the project entered the product-iteration phase
> documented in [`../CLAUDE.md`](../CLAUDE.md).
>
> The existing files are kept as a historical record and should not be
> read as the current state of the product — for that, see CLAUDE.md
> and the recent `git log`.

Each file captured: what was just shipped, what changed about the
working approach, what came next. Useful when re-reading the project
months later to understand a transition that the code alone doesn't
explain.
