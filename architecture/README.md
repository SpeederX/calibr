## Folder status

`design/` and `ux/` contain decisions and flows written during earlier product
phases. Their README files classify what is still current and what is retained
only as history. Historical documents explain why code exists; they do not
override current behavior.

The former strict-SemVer/spec/plan/memory methodology is retired. Branch and
workflow guidance lives only in `AGENTS.md`.
